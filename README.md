# Email-D-D-Template

A simple drag-and-drop email templater.

## Current product direction

The editor should open in a **true blank-state** experience so users can start from scratch.

- Do **not** pre-populate starter templates.
- Do **not** pre-populate starter fragments.
- Start with an empty canvas and empty saved libraries.

## Default app state

Use `config/default-state.json` as the source of truth for initial in-app data.
